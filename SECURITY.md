# Security Policy

## Supported Versions

We actively monitor and patch security vulnerabilities in the gateway. Please ensure you are running the latest stable release to receive security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes |
| < Latest| No |

Please upgrade immediately if you are running an older version and a security advisory is released.

## Reporting a Vulnerability

**Do not report security-related issues, vulnerabilities, or bugs containing sensitive information to the public issue tracker.** 

If you discover a security vulnerability within this project, please report it privately via email to ensure the safety of all deployments using this gateway.

### Disclosure Process

1. **Send an email** detailing the vulnerability to **AX1111YT@proton.me**.
2. Include as much detail as possible to help us reproduce and understand the scope of the issue:
   - A clear description of the vulnerability type (e.g., remote code execution, denial of service, injection).
   - Step-by-step instructions or a minimal proof of concept (PoC) to reproduce the flaw.
   - Any potential impact or configurations that make the vulnerability exploitable.
   - if possible, please attach a code fix
3. Do not disclose the issue publicly until we have had a reasonable amount of time to review, patch, and release a fix.

### Our Response Timeline

* **Acknowledgment:** You will receive an initial email acknowledgment within first few hours of your report.
* **Triage & Fix:** We will evaluate the report and work on a fix as quickly as possible. We will keep you updated on our progress.
* **Public Release:** Once a patch is ready, we will release a new version of the gateway along with a security advisory crediting you for the discovery (if you wish to be credited).
